GLSL ES Specification 1.00----4.5.2 精度限定符

返回目录

任何浮点或整数声明的类型前面都可以有以下精度限定符之一：

限定符 含义
highp 满足上述顶点语言的最低要求。在片段语言中是可选的。
mediump 满足上述对片段语言的最低要求。其范围和精度必须大于或等于lowp提供的范围和精度，小于或等于highp提供。
lowp 范围和精度可以小于mediump，但仍用于表示任何颜色通道的所有颜色值。

例如：

  lowp float color;
  varying mediump vec2 Coord;
  lowp ivec2 foo(lowp mat3);
  highp mat4 m;

精度限定符声明了底层在存储这些变量时必须使用的最小范围和精度。实现可能会使用比请求更大的范围和精度，但不会更小。用于实现特定精度限定符的范围和精度的增加量可能取决于变量、涉及变量的操作以及其他依赖于实现的细节。

精度限定符所需的最小范围和精度为

限定符 浮点范围 浮点幅度范围 浮点精度 整数范围
highp (-2⁶²,2⁶²) (2⁻⁶²,2⁶²) 相对：2⁻¹⁶ (-2¹⁶,2¹⁶)
mediump (-2¹⁴,2¹⁴) (2⁻¹⁴,2¹⁴) 相对：2⁻¹⁰ (-2¹⁰,2¹⁰)
lowp (−2,2) (2⁻⁸,2) 绝对：2⁻⁸ (-2⁸,2⁸)

其中浮点幅度范围是非零值的幅度范围。对于浮点精度，相对是指相对于该值测量的任何值的精度，适用于所有非零值。对于所有精度级别，必须精确表示零。对于整数类型，必须表示指定范围内的所有整数值。

如果实现无法为编译单元中变量的存储提供声明的精度，则必然会导致编译或链接错误。

对于高精度和中等精度，整数范围必须能够由同一精度限定符的相应浮点值准确表示。也就是说，highp int可以用highp float表示，mediump int也可以用mediump float表示。但是，lowp int不能用lowp float来表示；

顶点语言需要使用lowp、mediump和highp来编译和链接而不会出错。片段语言需要使用lowp和mediump才能编译而不会出错。支持highp是可选的。

实现提供的实际范围和精度可以通过API查询。

文字常量没有精度限定符。布尔变量同样如此。当所有构造函数参数都没有精度限定符时，浮点构造函数和整数构造函数也没有。

对于本段，“操作”包括运算符、内置函数和构造函数，“操作数”包括函数参数和构造函数参数。用于内部评估操作的精度，以及随后与任何中间值相关的精度限定，必须至少与操作所消耗的操作数的最高精度限定一样高。

对于未定义精度的常量表达式和子表达式，将以目标支持的最高精度（mediump或highp）或更高的精度进行计算。常量表达式的求值必须是不变的，通常在编译时执行。

在其他操作数没有精度限定符的情况下，精度限定符将来自其他操作数。如果没有操作数具有精度限定符，则将使用表达式中下一个消费操作的操作数的精度限定符。此规则可以递归应用，直到找到精度限定的操作数。如有必要，它还将包括赋值的l值、初始化器的声明变量、函数调用参数的形式参数或函数返回值的函数返回类型的精度限定。如果此方法无法确定精度，例如，如果整个表达式仅由没有精度限定符的操作数组成，并且结果未作为参数分配或传递，则以该类型的默认精度或更高精度进行计算。当在片段着色器中发生这种情况时，必须定义默认精度。

例如，考虑以下陈述

  uniform highp float h1;
  highp float h2 = 2.3 * 4.7; // 操作和结果精度高
  mediump float m;
  m = 3.7 * h1 * h2; // 所有操作都是高精度的
  h2 = m * h1; // 操作精度高
  m = h2 – h1; // 操作精度高
  h2 = m + m; // 中等精度的加法和结果
  void f(highp float p);
  f(3.3); // 3.3将以高精度传递

当浮点运算的结果大于（小于）所需精度可以存储的值时，结果可以是该精度可以表示的最大（最小）值，也可以是无穷大（负无穷大）的表示。例如，它不会导致包装行为、NaN的生成或异常情况。同样，如果结果比结果精度可以存储的值更接近零，则结果必须是零或（正确签名的）无穷小值的表示。

整数溢出导致未定义的值。它可能会包裹，也可能不会。

与其他限定符一样，精度限定符不会影响变量的基本类型。特别是，没有用于精度转换的构造函数；构造函数只转换类型。同样，与其他限定符一样，精度限定符不会导致基于参数类型的函数重载。正如下一章所讨论的，函数输入和输出是通过副本完成的，因此限定符不必匹配。
