GLSL ES Specification 1.00----4.3 存储限定符

返回目录

变量声明可能有一个存储限定符，在类型前面指定。这些总结如下

限定符 含义
< none: default > 本地读/写存储器或函数的输入参数
const 编译时常数或只读的函数参数
attribute 顶点着色器和OpenGL ES之间的链接，用于逐顶点数据
uniform 值在正在处理的图元之间不会改变，统一形成着色器、OpenGL ES和应用程序之间的链接
varying 插值数据的顶点着色器和片段着色器之间的链接

局部变量只能使用存储限定符const。

函数参数只能使用const存储限定符。参数限定符在第6.1.1节“函数调用约定”中有更详细的讨论。

函数返回类型和结构字段不使用存储限定符。

从着色器的一次运行到下一次运行（在片段之间或顶点之间进行通信）的通信数据类型不存在。这将阻止在多个顶点或片段上并行执行同一着色器。

没有存储限定符或只有const限定符的全局变量声明可能包含初始化器，在这种情况下，它们将在执行main()的第一行之前初始化。这样的初始化器必须是常量表达式。未在声明中或应用程序中初始化的没有存储限定符的全局变量将不会被OpenGL ES初始化，而是将使用未定义的值输入main()。统一、属性和变量可能没有初始化器。
