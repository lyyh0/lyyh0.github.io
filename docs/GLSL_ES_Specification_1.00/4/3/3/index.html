GLSL ES Specification 1.00----4.3.3 属性

返回目录

属性限定符用于声明从OpenGL ES按顶点传递给顶点着色器的变量。在除顶点着色器之外的任何类型的着色器中声明属性变量都是错误的。就顶点着色器而言，属性变量是只读的。属性变量的值通过OpenGL ES顶点API或作为顶点数组的一部分传递到顶点着色器。它们将顶点属性传递给顶点着色器，并且预计在每次顶点着色器运行时都会发生变化。属性限定符只能用于数据类型float、vec2、vec3、vec4、mat2、mat3、和mat4。属性变量不能声明为数组或结构。

声明示例：

  attribute vec4 position;
  attribute vec3 normal;
  attribute vec2 texCoord;

预计图形硬件将有少量固定位置用于传递顶点属性。因此，OpenGL ES着色语言将每个非矩阵属性变量定义为最多有四个浮点值的空间（即，一个vec4）。对可以使用的属性变量的数量有一个依赖于实现的限制，如果超过这个限制，将导致链接错误。（未使用的声明属性变量不计入此限制。）float属性与vec4在该限制下的计数相同，因此应用程序可能希望考虑将四个无关的float特性打包成一个vec4，以更好地利用底层硬件的功能。一个mat4属性将使用相当于4个vec4属性变量位置的值，一个mate3将使用等同于3个属性变量位置，一个mat2将使用2个属性变量地址。矩阵如何利用这个空间被实现通过API和语言隐藏。

属性变量必须具有全局作用域。
