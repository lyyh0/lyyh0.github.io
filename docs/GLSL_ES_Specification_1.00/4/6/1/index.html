GLSL ES Specification 1.00----4.6.1 不变限定符

返回目录

为了确保特定的输出变量是不变的，有必要使用invariant限定符。它可以用来将之前声明的变量限定为不变变量

  invariant gl_Position; // 使现有的gl_Position保持不变
  varying mediump vec3 Color;

  invariant Color; // 使现有颜色不变

或者在声明变量时作为声明的一部分

  invariant varying mediump vec3 Color;

这些是唯一可以使用不变限定符的情况。当与声明结合时，不变限定符必须出现在任何存储限定符（varying）之前。

只有以下变量可以声明为不变量：

  • 从顶点着色器输出的内置特殊变量

  • 改变顶点着色器输出的变量

  • 输入到片段着色器的内置特殊变量

  • 改变输入到片段着色器的变量

  • 片段着色器输出的内置特殊变量。

invariant关键字后面可以跟一个逗号分隔的先前声明的标识符列表。所有对invariant的使用都必须在全局范围内，并且在将变量声明为不变量之前。

为了保证特定输出变量在两个着色器中的不变性，以下条件也必须成立：

  • 输出变量在两个着色器中都声明为不变的。

  • 表达式和控制流消耗的所有着色器输入变量都必须输入相同的值，这些值有助于分配给输出变量的值。

  • 对于任何有助于输出变量值的纹理函数调用，纹理格式、纹理像素值和纹理过滤的设置方式都是相同的。

  • 所有输入值都以相同的方式操作。重要表达式和任何中间表达式中的所有操作必须相同，具有相同的操作数顺序和相同的结合性，才能给出相同的求值顺序。中间变量和函数必须声明为具有相同显式或隐式精度限定符的相同类型。任何影响输出值的控制流都必须相同，用于确定此控制流的任何表达式也必须遵循这些不变性规则。

本质上，所有导致不变输出的数据流和控制流都必须匹配。

最初，默认情况下，所有输出变量都允许是变量。要强制所有输出变量保持不变，请使用pragma

  #pragma STDGL invariant(all)

在编译单元中的所有声明之前。如果在声明任何变量或函数之后使用此语法，则表现为不变的输出集是未定义的。

通常，不变性是以牺牲优化的灵活性为代价来确保的，因此使用不变性可能会降低性能。因此，使用此语法是为了辅助调试，避免单独将所有输出变量声明为不变量。
