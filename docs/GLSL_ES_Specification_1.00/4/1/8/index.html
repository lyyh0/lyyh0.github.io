GLSL ES Specification 1.00----4.1.8 结构

返回目录

用户定义的类型可以通过使用struct关键字将其他已定义的类型聚合到一个结构中来创建。例如，

  struct light {
    float intensity;
    vec3 position;
  } lightVar;

在这个例子中，light成为新类型的名称，lightVar成为light类型的变量。要声明新类型的变量，请使用其名称（不带关键字struct）。

  light lightVar2;

更正式地说，结构声明如下。然而，完整正确的语法如第9节“着色语言语法”所示。

  struct-definition :
    qualifiersopt struct nameopt { member-list } declaratorsopt ;

  member-list :
    member-declaration;
    member-declaration member-list;

  member-declaration :
    basic-type declarators;

其中name成为用户定义的类型，可用于将变量声明为这种新类型。该名称与其他变量、类型和函数共享相同的命名空间。所有以前可见的具有该名称的变量、类型、构造函数或函数都是隐藏的。可选的qualifiers仅适用于任何declarators，而不是为name定义的类型的一部分。

结构必须至少有一个成员声明。成员声明符可以包含精度限定符，但不能包含任何其他限定符。不支持位字段。成员类型必须已经定义（没有正向引用，也没有嵌入定义）。成员声明不能包含初始化器。成员声明符可以包含数组。此类数组必须指定大小，并且大小必须是大于零的积分常数表达式（见第4.3.3节“积分常数表达式”）。每一级结构都有自己的命名空间，用于成员声明器中给出的名称；这样的名称只需要在该名称空间内是唯一的。

不支持匿名结构声明器（类型为结构但没有声明器的成员声明）。

  struct S
  {
    int x;
  };
  struct T
  {
    S; // 错误：不允许使用匿名结构.
    int y;
  };

不支持嵌入式结构定义：

示例1：

  struct S
  {
    struct T // 错误：不支持嵌入式结构定义。
    {
      int a;
    } t;
  int b:
  };

示例2：

  struct T
  {
    int a;
  };
  struct S
  {
    T t; // ok.
    int b;
  };
结构可以在声明时使用构造函数初始化，如第5.4.3节“结构构造函数”所述。

对类型或限定符使用的任何限制也适用于包含该类型或限定词的结构。这是递归应用的。
