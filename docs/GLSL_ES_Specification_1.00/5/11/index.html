GLSL ES Specification 1.00----5.11 向量和矩阵运算

返回目录

除少数例外情况外，操作都是按组件进行的。当运算符对向量或矩阵进行操作时，它以分量方式独立地对向量或基质的每个分量进行操作。例如，

  vec3 v, u;
  float f;

  v = u + f;

相当于：

  v.x = u.x + f;
  v.y = u.y + f;
  v.z = u.z + f;

和

  vec3 v, u, w;
  w = v + u;

相当于：

  w.x = v.x + u.x;
  w.y = v.y + u.y;
  w.z = v.z + u.z;

对于大多数运算符以及所有整数、浮点向量和矩阵类型也是如此。例外情况是矩阵乘以向量、向量乘以矩阵和矩阵乘以矩阵。这些操作不是按分量进行的，而是执行正确的线性代数乘法。它们要求操作数的大小匹配。

  vec3 v, u;
  mat3 m;

  u = v * m;

等效于

  u.x = dot(v, m[0]); // m[0]是m的左列
  u.y = dot(v, m[1]); // dot(a,b)是a和b的内(dot)积
  u.z = dot(v, m[2]);

和

  u = m * v;

等效于

  u.x = m[0].x * v.x + m[1].x * v.y + m[2].x * v.z;
  u.y = m[0].y * v.x + m[1].y * v.y + m[2].y * v.z;
  u.z = m[0].z * v.x + m[1].z * v.y + m[2].z * v.z;

和

  mat m, n, r;

  r = m * n;

等效于

  r[0].x = m[0].x * n[0].x + m[1].x * n[0].y + m[2].x * n[0].z;
  r[1].x = m[0].x * n[1].x + m[1].x * n[1].y + m[2].x * n[1].z;
  r[2].x = m[0].x * n[2].x + m[1].x * n[2].y + m[2].x * n[2].z;

  r[0].y = m[0].y * n[0].x + m[1].y * n[0].y + m[2].y * n[0].z;
  r[1].y = m[0].y * n[1].x + m[1].y * n[1].y + m[2].y * n[1].z;
  r[2].y = m[0].y * n[2].x + m[1].y * n[2].y + m[2].y * n[2].z;

  r[0].z = m[0].z * n[0].x + m[1].z * n[0].y + m[2].z * n[0].z;
  r[1].z = m[0].z * n[1].x + m[1].z * n[1].y + m[2].z * n[1].z;
  r[2].z = m[0].z * n[2].x + m[1].z * n[2].y + m[2].z * n[2].z;

对于大小为2和4的向量和矩阵也是如此。

所有一元运算都在其操作数上按组件工作。对于二元算术运算，如果两个操作数的类型相同，则该运算是按组件进行的，并产生与操作数类型相同的结果。如果一个操作数是标量浮点数，另一个是向量或矩阵，则操作将继续进行，就像复制标量值以形成匹配的向量或矩阵操作数一样。
