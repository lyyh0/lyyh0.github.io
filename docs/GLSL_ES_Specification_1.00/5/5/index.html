GLSL ES Specification 1.00----5.5 向量分量

返回目录

向量各分量的名称用一个字母表示。为了便于符号表示，根据位置、颜色或纹理坐标矢量的常见用法，每个组件都有几个字母。向量的各个分量可以通过在变量名后面加句点（.），然后再加上分量名来选择。

支持的组件名称包括：
{x, y, z, w} 在访问表示点或法线的向量时很有用
{r, g, b, a} 在访问表示颜色的向量时很有用
{s, t, p, q} 在访问表示纹理坐标的向量时很有用

例如，组件名称x、r和s是向量中相同（第一个）组件的同义词。

请注意，OpenGL ES中纹理的第三个分量r已重命名为p，以避免与颜色中的r（代表红色）混淆。

访问超出为向量类型声明的组件是一个错误，例如：

  vec2 pos;
  pos.x // 是合法的
  pos.z // 是非法的

组件选择语法允许通过在句点（.）后附加名称（来自同一名称集）来选择多个组件。

  vec4 v4;
  v4.rgba; // 是一个vec4，仅与使用v4相同，
  v4.rgb; // 是vec3，
  v4.b; // 是浮点数，
  v4.xy; // 是vec2，
  v4.xgba; // 是非法的 - 组件名称不来自同一组。

最多只能选择4个组件。

  vec4 v4;
  v4.xyzw; // 是一个vec4
  v4.xyzwxy; // 是非法的，因为它有6个组件
  (v4.xyzwxy).xy; // 是非法的，因为中间值有6个分量

  vec2 v2;
  v2.xyxy; // 是合法的。它的计算结果为vec4。

组件的顺序可以不同，以搅拌或复制它们：

  vec4 pos = vec4(1.0, 2.0, 3.0, 4.0);
  vec4 swiz= pos.wzyx; // swiz = (4.0, 3.0, 2.0, 1.0)
  vec4 dup = pos.xxyy; // dup = (1.0, 1.0, 2.0, 2.0)

这种表示法比构造函数语法更简洁。为了形成r值，它可以应用于任何产生向量r值的表达式。

组件组符号可以出现在表达式的左侧。

  vec4 pos = vec4(1.0, 2.0, 3.0, 4.0);
  pos.xw = vec2(5.0, 6.0); // pos = (5.0, 2.0, 3.0, 6.0)
  pos.wx = vec2(7.0, 8.0); // pos = (8.0, 2.0, 3.0, 7.0)
  pos.xx = vec2(3.0, 4.0); // 非法 - ‘x’使用了两次
  pos.xy = vec3(1.0, 2.0, 3.0); // 非法 - vec2和vec3之间不匹配

要形成l值，必须对向量类型的l值应用swizzling，不包含重复的分量，并且根据指定的分量数量，它会导致标量或向量类型的l值。

数组下标语法也可以应用于向量，以提供数字索引。所以在

  vec4 pos;

pos[2]是pos的第三个元素，等价于pos.z。这允许对向量进行变量索引，以及访问组件的通用方法。任何整数表达式都可以用作下标。第一个组件位于索引零处。使用值为负或大于等于向量大小的常数积分表达式从向量读取或向向量写入是非法的。当使用非常量表达式进行索引时，如果索引为负或大于或等于向量的大小，则行为未定义。
